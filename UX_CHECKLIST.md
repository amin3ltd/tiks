# pretix E2E UX Checklist

## Admin experience
- [ ] Admin login is easy to find and works with valid credentials
- [ ] Dashboard loads quickly and clearly shows next actions
- [ ] Organizer list is visible, searchable, and actionable
- [ ] Creating or verifying an organizer is straight-forward
- [ ] Admin can reach organizer/event overview without confusion

## Organizer experience
- [ ] Organizer login takes you to a clear organizer control panel
- [ ] Event creation form uses understandable labels and default values
- [ ] Event name, slug, date, timezone, and location are easy to enter
- [ ] Ticket product creation is clear for regular/VIP/free tickets
- [ ] Pricing, quotas, availability, and sale dates are visible and editable
- [ ] Publishing/going live is communicated clearly
- [ ] Public event page is visible and understandable

## Customer experience
- [ ] Public event page is easy to discover and read on desktop/mobile
- [ ] Ticket quantity selection is obvious and accessible
- [ ] Checkout steps are clearly labeled and have progress feedback
- [ ] Required fields are clearly marked and validated
- [ ] Errors show meaningful messaging for invalid email/missing fields
- [ ] Payment instructions are easy to understand
- [ ] Confirmation page clearly shows order code and ticket download options
- [ ] Ticket download/QR is easy to find and use

## Order management and check-in
- [ ] Orders list/search is easy to use for reference or code lookup
- [ ] Manual payments can be marked paid with clear feedback
- [ ] Resending ticket email is easy to locate
- [ ] Check-in list creation/selection is straightforward
- [ ] Searching attendees/orders is fast and intuitive
- [ ] Duplicate check-in is blocked or shows a clear warning
- [ ] Attendance count updates after check-in

## Mobile experience
- [ ] Customer flow works on a phone-sized viewport
- [ ] Organizer flow is legible on mobile or responsive layout
- [ ] Check-in staff can use mobile search/screen scanning flow

## Non-functional checks
- [ ] No 500/403 pages appear during the flow
- [ ] Key pages have expected text and headings
- [ ] Site responds within a reasonable delay for each major interaction
